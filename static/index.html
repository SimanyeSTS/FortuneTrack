Please see below tables for FortuneTrack API defined endpoints.



Home

HTTP Methods Path Description Remark
GET /(API Home Documentation Page) This is the home page of the FortuneTrack Restful API. It contains information about the API and it's endpoints. -



Users

HTTP Methods Path Description Remark
GET /users This endpoint when used retrieves all the registered users and their details from the database. -
GET /user/:id This endpoint is used to retrieve a specific user and their details according to their UserID from the database. -
POST /user/register This endpoint is used to register a new user to the database and FortuneTrack. -
POST /user/login This endpoint is used to login a registered user/admin to FortuneTrack using their emailAdd and userPass. -
PATCH/PUT /user/:id This endpoint is used to update a registered user's details according to their UserID from the database. -
DELETE /user/:id This endpoint is used to remove a registered user and their details according to their UserID from the database. -



All Predictions

HTTP Methods Path Description Remark
GET /from/db/all-predictions This endpoint when used retrieves all the predictions from the database. -



Retail

HTTP Methods Path Description Remark
GET /from/api/retail This endpoint when used retrieves the default retail prediction from the AlphaVantage API and saves it to the database. Please note, running this endpoint is not recommended as it will result in the duplication of the default retail prediction. Cron Job has been setup to automatically update the prediction data as needed.
GET /from/db/retail-data This endpoint is used to retrieve all the retail predictions from the database. -
GET /from/db/retail/:id This endpoint is used to retrieve a specific retail prediction according to it's id from the database. -
PUT/PATCH /from/db/retail/:id This endpoint is used to update a retail prediction's details according to it's id from the database. -
POST /to/db/retail-data This endpoint is used to manually add a retail prediction to the database. This is not recommended as it could result in false predictions.
DELETE /from/db/retail/:id This endpoint is used to remove a retail prediction according to it's id in the database. -
